A shared brain is a memory your team builds together. It might be the company knowledge base, a client project, a product area, or anything else that more than one person needs to find and add to. Unlike your Personal brain, a shared brain is created on purpose, shared on purpose, and written to on purpose, and those three "on purposes" capture everything important about how it works.

## Nothing arrives by accident

The most important thing to understand about shared brains is what does *not* happen. Nothing flows into them automatically. There is no setting that routes your captures to a shared brain by default. An agent connected to your Personal brain cannot write to a shared brain, because a Connection physically reaches only the brain it was approved for and cannot reach anywhere else.

For something to appear in a shared brain, someone with Write access must deliberately put it there: by working directly in that brain's Inbox, through an agent Connection approved specifically for that brain, or by [promoting](/brains/promoting) something across from their Personal brain.

<Note>
This deliberate-only model is what keeps shared brains trustworthy. When you find a Page in a shared brain, you can be confident a person or an agent acting on a person's instruction put it there on purpose.
</Note>

## Members and roles

People are added to a shared brain with one of four roles. Each role builds on the one above it.

| Role | What they can do |
|---|---|
| **Read** | Browse Pages, search the brain, and read the Inbox. Cannot make any changes. |
| **Write** | Everything a Read member can do, plus add items to the Inbox, file items, and create or edit Pages. |
| **Admin** | Everything a Write member can do, plus manage members, manage Connections, and change brain settings. |
| **Owner** | Everything an Admin can do, plus permanently delete the brain. Every shared brain has exactly one Owner. |

Agent Connections carry a role too. When you connect an agent to a shared brain, the approval screen shows the access level that Connection will have. A Connection can never do more than the person who approved it: if you hold a Write role, you cannot approve a Connection with Admin access.

<Tip>
If you are setting up a shared brain for a team, start by giving most members Write access. Reserve Admin for the people who will manage membership, and keep Owner for the person responsible for the brain long-term.
</Tip>

## Creating a shared brain

<Steps>
  <Step title="Open Brains and create a new brain">
    From the **Brains** section of Cortex, choose to create a new brain. Give it a clear name that reflects its purpose. "Team Handbook", "Client: Acme Ltd", or "Q3 Strategy" work better than generic names.
  </Step>
  <Step title="Pick a template">
    Choose a [template](/brains/templates) that matches what the brain is for. A template gives the brain its starting folder structure and filing guidance. You can always add to it later.
  </Step>
  <Step title="Invite members">
    Add the people who need access and assign each one a role. The brain starts empty; its Inbox works exactly like the one in your Personal brain.
  </Step>
</Steps>

## Try it

1. Create a shared brain called "Team Handbook" using the Company template.
2. Invite a teammate with the Write role.
3. Drop a document into its Inbox and watch it become the brain's first Page.

title: "Uploading files"

The simplest way to put something in a brain is to upload it. Open the brain's **Inbox** and drag a file in, or use the upload button. It arrives as a new inbox item with its text extracted, ready to be filed.

## What you can upload

| Format | What happens |
|---|---|
| Markdown and plain text | Used as-is. |
| PDF | The text is extracted, including from multi-page documents. |
| Word (`.docx`) | The document's text is extracted. |
| PowerPoint (`.pptx`) | The text from every slide is extracted. |

Each file can be up to 50 MB. Whatever you upload, Cortex pulls out the words so the content becomes searchable, even a slide deck becomes text your agent can recall.

## What extraction does, and does not, do

Extraction reads the words. It does not try to interpret images, charts, or the visual layout of a page. If a crucial fact only exists as a picture, it will not be captured. For anything text-based, extraction is reliable.

## Duplicates

Upload the same file twice and Cortex recognises it rather than making a second copy. It points you back to the item you already have. This means you can re-upload without worrying about clutter. [More on how duplicates are handled.](/capture/the-inbox#duplicates-take-care-of-themselves)

## Try it

1. Open a brain's Inbox and drag in a PDF.
2. Open the new item and read the extracted text.
3. Drag the same PDF in again and watch Cortex point you back to the first item.

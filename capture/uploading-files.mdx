The simplest way to put something in a brain is to upload it. Open the brain's **Inbox**, drag a file in, or use the upload button. It arrives as a new inbox item with its text already extracted, ready to be filed into a page.

## Supported formats

| Format | What Cortex does with it |
|---|---|
| Markdown and plain text | Used as-is, no conversion needed. |
| PDF | Text is extracted, including across multi-page documents. |
| Word (`.docx`) | The document's text is extracted. |
| PowerPoint (`.pptx`) | Text from every slide is extracted. |

Each file can be up to 50 MB. Whatever you upload, Cortex pulls out the words so the content becomes searchable and your agent can recall it.

## What extraction does, and does not, do

Extraction reads the words on the page. It does not interpret images, charts, or the visual layout of a document. If a crucial fact exists only as a picture, a diagram, or a chart label, it will not be captured. For anything text-based, extraction is reliable.

<Note>
If your document relies heavily on visuals to convey information, consider adding a plain-text summary to the file before uploading, or creating a brief note page in the brain that describes what the visual shows.
</Note>

## Duplicates

Upload the same file twice and Cortex recognises it rather than creating a second copy. It points you back to the item you already have. This means you can re-upload without worrying about clutter.

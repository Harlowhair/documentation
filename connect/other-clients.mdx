Cortex connects over the Model Context Protocol (MCP), an open standard for giving AI tools access to external systems. Any MCP-capable client can connect to a Cortex brain -- not just the ones with their own dedicated guide. If your tool can add a remote MCP server by URL, the steps below apply to it.

## The general shape

Every Cortex connection works the same way, regardless of which client you are using:

1. **The endpoint** is your brain's URL, `https://api.oncortex.ai/mcp/your-brain`, shown on the brain's Connect page under the generic tab. It uses the standard streamable HTTP transport.
2. **Sign-in is automatic.** Cortex advertises how to authenticate using standard MCP discovery, so any client with sign-in support will send you to Cortex to approve the connection, then connect itself. There are no API keys to paste for interactive use.
3. **One connection reaches one brain.** As with every Cortex connection, add a separate connection for each brain you want the client to use.

## Cursor

Cursor supports MCP servers in its settings. Add a new MCP server using the Cortex connection URL from the brain's Connect page. When Cursor first uses it, approve the connection in the browser. After that, Cursor can search and file into that brain during any session.

## Anything else

For any other MCP-capable client, the recipe is the same: give it your brain's connection URL, let it walk you through the browser approval, and you are connected.

If your client cannot complete the browser sign-in step (for example, an automated script or a server with no human present), use a service credential instead. A service credential is a static key you create yourself and paste directly into the automation.

## Compatibility

MCP client support is moving quickly. The reliable test is simple: if your tool can add a remote or custom MCP connector by URL and supports browser sign-in, it works with Cortex. If it can only use static keys, connect it as an automation with a service credential.
